# 🏫 선문대 동아리 관리 및 홍보 게시판 시스템 (C Language)

선문대학교 학과 구조를 기반으로 한 **C 언어 기반의 동아리 회원 관리 및 맞춤형 홍보 게시판 프로그램**입니다. 
정수형 전공 코드를 도입하여 문자열 입력 오타로 인한 버그를 원천 차단하고, 동아리장과 일반 회원 간의 명확한 권한 분리를 통해 유기적인 동아리 운영 비즈니스 로직을 구현했습니다.

---

## 🚀 주요 핵심 매커니즘

### 1. 선문대 맞춤형 전공 코드 시스템
입력 오류 및 문자열 불일치(`strcmp` 예외)를 방지하기 위해 단과대와 세부 학과를 **정수형(int) 코드**로 그룹화하여 관리합니다.
* **공과대학/SW융합대학 (Code 1):** 컴퓨터공학부(1), AI소프트웨어학과(2), 전자공학과(3), 기계공학과(4)
* **인문사회/경영대학 (Code 2):** 경영학과(1), 미디어커뮤니케이션학과(2), 상담심리학과(3)
* **보건의료대학 (Code 3):** 간호학과(1), 물리치료학과(2), 치위생학과(3)

### 2. 전공 동아리 가입 제한 기능 (방향성 채택)
* **조회 권한:** 모든 사용자가 전공 동아리 홍보글을 자유롭게 조회하고 탐색할 수 있습니다. (메뉴 목록에 `[공과대학-컴퓨터공학부]`와 같은 태그 노출)
* **가입 권한:** 가입 신청 시 사용자의 전공 코드와 동아리의 전공 코드를 비교하여 **완벽히 일치하는 경우에만 신청 접수가 가능**하도록 제한합니다.

---

## ✨ 주요 기능 명세

### 👤 1. 마이페이지 및 회원 관리
* **정보 조회:** 내 기본 정보와 내가 가입한 동아리 목록, 그리고 각 동아리 내에서의 권한(`MEMBER` / `OWNER`)을 한눈에 확인합니다.
* **프로필 수정:** 현재 비밀번호 인증 후 **닉네임**과 **비밀번호**만 수정 가능합니다. (전체 유저 대상 닉네임 중복 검사 로직 포함)
* **계정 탈퇴:** * 유저가 어느 동아리의 **동아리장(OWNER)**일 경우 탈퇴가 제한되며, 위임 및 폐쇄 안내를 출력합니다.
  * 일반 회원으로 가입된 동아리가 있다면 탈퇴 시 해당 동아리 멤버 명단에서 자동으로 선삭제 처리됩니다.

### 📢 2. 홍보 게시판 & 상호작용
* **게시글 목록:** 목록 화면에서부터 `[단과대-학과명]` 태그를 명시하여 직관적인 필터링 화면을 제공합니다. (교양 동아리는 `[교양]` 태그 노출)
* **상세 보기:** 게시글 ID(번호)를 선택하여 상세 내용, 작성자 정보, 소속 동아리명, 그리고 댓글 목록을 조회합니다.
* **상호작용:** 게시글 좋아요 증가, 댓글 작성, 그리고 **댓글별 좋아요 증가** 액션을 지원합니다.

### 👑 3. 동아리장 전용 관리 기능
* **가입 신청 승인 페이지:** 대기(`PENDING`) 중인 신청자 목록을 확인하고, 선택한 학생을 **승인(`APPROVED`)**하여 멤버로 추가하거나 **거절(`REJECTED`)**할 수 있습니다.
* **동아리 관리:** 동아리 정보(이름, 소개 글) 수정 기능 및 타 회원에게 동아리장 권한을 넘겨주는 **동아리장 위임 기능**을 지원합니다. (위임 시 본인은 일반 회원으로 변경)

### 📬 4. 독립 메시지함 (공지 & 쪽지)
* **단체 공지 발송:** 동아리장이 공지를 작성하면 해당 동아리에 소속된 모든 멤버의 메시지함으로 일괄 전송됩니다.
* **1:1 쪽지:** 상대방의 유저 ID를 검색하여 쌍방향 쪽지를 보낼 수 있습니다. (존재하지 않는 ID 예외 처리)
* **조회 화면:** `[공지]`와 `[쪽지]` 머리말을 달아 수신된 메시지를 명확히 구분하여 출력합니다.

## 🛠️ 데이터 구조 (Data Structures)

프로그램 내에서 데이터 연동을 위해 사용되는 핵심 구조체 설계 구조입니다.

```c
// 권한 및 상태 관리를 위한 Enum
typedef enum { MEMBER, OWNER } Role;
typedef enum { PENDING, APPROVED, REJECTED } RequestStatus;
typedef enum { ANNOUNCEMENT, DIRECT_MESSAGE } MsgType;

// 회원 정보 구조체
typedef struct {
    char student_id[20];   // 학번 (고유 ID)
    char password[30];
    char name[20];
    char nickname[30];
    int college_code;      // 단과대 코드
    int major_code;         // 세부 전공 코드
} User;

// 홍보 게시글 및 댓글 구조체
typedef struct {
    int comment_id;
    char writer_id[20];
    char content[100];
    int likes;             // 댓글 좋아요 수
} Comment;

typedef struct {
    int post_id;
    int club_id;
    char title[50];
    char content[500];
    char author_name[20];
    int likes;             // 게시글 좋아요 수
    Comment comments[50];
    int comment_count;
} Post;
